# ⏰ Scheduling

## Overview

Scheduling allows Make.com scenarios to run automatically at specific times or intervals without manual intervention. It ensures your automations execute consistently based on your business requirements.

---

## What is Scheduling?

Scheduling defines when and how often a scenario runs.

Scheduling allows you to:

- Run automations automatically
- Execute workflows at fixed intervals
- Schedule recurring tasks
- Reduce manual work
- Maintain consistent operations

---

## Scheduling Options

Make.com supports several scheduling methods:

- Immediately
- At Regular Intervals
- Hourly
- Daily
- Weekly
- Monthly
- Custom Schedule

---

## Typical Workflow

1. Create a scenario.
2. Configure all modules.
3. Test the workflow.
4. Choose a scheduling option.
5. Set the execution frequency.
6. Enable the scenario.
7. Monitor scheduled runs.

## Benefits

- Fully automated execution
- Consistent workflow timing
- Reduced manual effort
- Increased productivity
- Reliable task automation

---

## Best Practices

- Choose an appropriate schedule.
- Test before enabling automation.
- Avoid unnecessary frequent executions.
- Monitor execution history regularly.
- Update schedules as business needs change.

---

## Common Mistakes

- Running scenarios too frequently.
- Forgetting to enable scheduling.
- Ignoring execution failures.
- Scheduling overlapping workflows.
- Not monitoring automation performance.

---

## Summary

Scheduling enables Make.com to run workflows automatically at the right time, helping you build reliable, efficient, and hands-free automations.
