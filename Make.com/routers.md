# 🔀 Routers

## Overview

Routers allow a Make.com scenario to split into multiple execution paths. They enable a single workflow to perform different actions based on conditions, making automations more flexible and efficient.

---

## What are Routers?

A Router creates multiple branches within a scenario.

Routers allow you to:

- Split workflows
- Process different data paths
- Run multiple actions
- Build conditional logic
- Create advanced automations

---

## How Routers Work

A router receives data from a module and sends it to one or more branches.

Each branch can:

- Execute independently
- Use different modules
- Apply different filters
- Perform separate actions

---

## Common Use Cases

- Customer Segmentation
- Order Processing
- Multi-channel Notifications
- AI Content Workflows
- Marketing Automation
- Support Ticket Routing
- Data Processing Pipelines

---

## Typical Workflow

1. Add a Router after a module.
2. Create multiple branches.
3. Add modules to each branch.
4. Apply filters where needed.
5. Test every branch.
6. Activate the scenario.

---

## Benefits

- Flexible workflow design
- Multiple execution paths
- Better organization
- Reduced duplication
- Scalable automations

---

## Best Practices

- Keep each branch focused on one task.
- Use filters to control execution.
- Clearly label workflow branches.
- Test every route individually.
- Remove unused branches.

---

## Common Mistakes

- Creating unnecessary branches.
- Forgetting to add filters.
- Building overly complex routing.
- Ignoring branch testing.
- Duplicating identical workflows.

---

## Summary

Routers make Make.com workflows more powerful by allowing automations to split into multiple paths and execute different actions based on your business logic.
