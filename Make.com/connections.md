# 🔗 Connections

## Overview

Connections allow Make.com to securely communicate with external applications and services. Before a scenario can exchange data, you must create and authorize a connection.

---

## What are Connections?

A connection is a secure link between Make.com and an external application.

Connections enable you to:

- Access application data
- Send information
- Receive updates
- Trigger automations
- Authenticate securely

---

## Types of Connections

Common connection types include:

- OAuth Connections
- API Key Connections
- Basic Authentication
- Bearer Token Authentication
- Custom API Connections

---

## Typical Workflow

1. Select an application.
2. Create a new connection.
3. Sign in to the application.
4. Grant the required permissions.
5. Save the connection.
6. Test the connection.
7. Use it in your scenario.

---

## Common Use Cases

- Google Workspace

---

## Benefits

- Secure authentication
- Easy integration
- Reusable connections
- Fast setup
- Centralized management

---

## Best Practices

- Use descriptive connection names.
- Enable two-factor authentication (2FA) where available.
- Remove unused connections.
- Review permissions regularly.
- Store API credentials securely.

---

## Common Mistakes

- Using expired credentials.
- Granting unnecessary permissions.
- Creating duplicate connections.
- Sharing API keys.
- Forgetting to reconnect expired accounts.

---

## Summary

Connections provide the secure foundation for every Make.com automation by enabling authenticated communication with external applications and services.
