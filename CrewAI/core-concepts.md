# 🧠 Core Concepts

## Overview

CrewAI is built around a few fundamental concepts that enable multiple AI agents to collaborate efficiently. Understanding these concepts is essential before building complex workflows.

---

## Agents

Agents are specialized AI workers with defined roles, goals, and responsibilities. Each agent focuses on a specific part of a workflow.

Examples:

- Research Agent
- Writer Agent
- Reviewer Agent
- Coding Agent

---

## Tasks

Tasks define the work assigned to an agent. Every task has a clear objective and expected outcome.

Examples:

- Research a topic
- Write an article
- Review code
- Summarize documents

---

## Crews

A Crew is a team of agents working together to complete a shared objective. Each agent contributes its expertise to achieve the final result.

---

## Tools

Tools extend an agent's capabilities by allowing it to access external services, APIs, databases, search engines, or custom functions.

---

## Processes

Processes define how tasks are executed within a crew, such as sequential execution or collaborative workflows.

---

## Memory

Memory allows agents to remember useful information during or across workflows, improving consistency and reducing repeated work.

---

## Knowledge

Knowledge sources provide agents with reliable information from documents, databases, or external resources to improve decision-making.

---

## Workflow

A typical CrewAI workflow looks like this:

1. Create Agents
2. Assign Tasks
3. Build a Crew
4. Select a Process
5. Execute the Workflow
6. Review Results

---

## Benefits

- Modular architecture
- Better collaboration
- Reusable agents
- Scalable workflows
- Easier maintenance

---

## Summary

The core concepts of CrewAI—Agents, Tasks, Crews, Tools, Processes, Memory, and Knowledge—work together to build intelligent, collaborative AI systems.
