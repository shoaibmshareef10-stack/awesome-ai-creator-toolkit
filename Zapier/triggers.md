# 🚀 Triggers

## Overview

Triggers are the starting point of every Zap in Zapier. A Trigger listens for a specific event in an application and automatically starts the workflow when that event occurs.

---

## What are Triggers?

A Trigger is an event that activates a Zap.

Triggers can:

- Detect new data
- Monitor application events
- Start workflows automatically
- Receive updates
- Launch automated processes

---

## Common Trigger Types

Zapier supports many trigger events, including:

- New Record
- New Email
- New Form Submission
- New Order
- New Contact
- Updated Record
- Scheduled Trigger

---

## Typical Workflow

1. Create a new Zap.
2. Select the Trigger app.
3. Choose a Trigger event.
4. Connect your account.
5. Test the Trigger.
6. Continue building the Zap.

## Best Practices

- Choose the correct Trigger event.
- Test the Trigger before publishing.
- Use descriptive Zap names.
- Verify connected accounts.
- Monitor Trigger performance regularly.

---

## Common Mistakes

- Selecting the wrong Trigger.
- Ignoring Trigger test results.
- Using unnecessary polling.
- Forgetting account permissions.
- Not monitoring failed executions.

---

## Summary

Triggers are the starting point of every Zap, automatically detecting events and launching workflows without manual intervention.
